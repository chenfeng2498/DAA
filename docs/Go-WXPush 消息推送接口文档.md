## 当前测试结论

- 接口已完成连通性测试。
- 使用默认接收用户发送测试消息成功。
- 当前项目后续价格提醒、系统告警等通知可直接调用该接口。

## 接口地址

```
http://111.230.16.149:5566/wxsend
```

## 请求方式

- GET / POST（推荐 GET，简单方便）

## 请求参数

支持通过 Query String 或 JSON Body 传参。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `title` | string | ✅ 是 | 消息标题（如：服务器报警） |
| `content` | string | ✅ 是 | 消息正文内容 |
| `userid` | string | ❌ 否 | 接收用户的 OpenID，不传则发给默认用户（已配置） |
| `template_id` | string | ❌ 否 | 模板 ID，不传则使用默认模板（已配置） |

> **默认配置**：已绑定测试号，默认接收用户为 `oQEbY277xNWNfWEMnTDuIg9e_ayQ`，默认模板 ID 为 `yFMKEFTTAPwGIiG7A8yOW2u9lb_RLN-P35SjIVhhGnM`。

## 调用示例

### 1. 最简单的 GET 请求（发给默认用户）

```bash
curl "http://111.230.16.149:5566/wxsend?title=测试消息&content=服务运行正常"
```

### 2. 发给指定用户

```bash
curl "http://111.230.16.149:5566/wxsend?title=私人提醒&content=记得开会&userid=目标用户的OpenID"
```

### 3. 使用自定义模板

```bash
curl "http://111.230.16.149:5566/wxsend?title=通知&content=内容&template_id=其他模板ID"
```

### 4. Python 调用示例

```python
import requests

url = "http://111.230.16.149:5566/wxsend"
params = {
    "title": "服务器告警",
    "content": "CPU 使用率 95%"
}
resp = requests.get(url, params=params)
print(resp.json())  # 成功返回 {"errcode":0,"errmsg":"ok"}
```

## 返回结果说明

| 返回字段 | 说明 |
|---------|------|
| `errcode` | 0 表示成功，非0表示失败（详见微信官方错误码） |
| `errmsg` | 错误描述，成功时为 "ok" |

常见错误：
- `40003`：无效的 OpenID（用户未关注测试号或 ID 错误）
- `41001`：缺少 access_token（通常为 secret 配置错误）
- `45015`：回复时间超过限制（一般不会出现）

## 项目接入建议

- 后端触发价格提醒或系统告警时，直接向 `http://111.230.16.149:5566/wxsend` 发起 GET 请求。
- `title` 建议使用固定业务前缀，例如 `大A监控价格提醒`、`大A监控系统告警`。
- `content` 建议包含股票名称、代码、当前价格、目标价格、触发时间和提醒类型，便于用户在微信中直接判断。
- 未传 `userid` 时发送给默认测试用户；正式多人使用时再扩展用户 OpenID 配置。
- 调用失败时不要把业务提醒标记为已完成，应记录 `errcode`、`errmsg` 并等待下一轮重试。
