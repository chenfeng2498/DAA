本项目的 `backend/server.py` 是一体化 HTTP 服务：

- 提供前端静态页面，例如 `index.html`

因此线上环境不需要再单独运行 `python3 -m http.server 8000`。如果只启动静态服务，页面可以打开，但 `/api/dashboard` 会无法访问，页面数据不会显示。

## 4. 首次部署流程

### 4.1 安装基础依赖

服务器预装了 Python 3 和 Git。项目当前唯一明确需要额外安装的 Python 依赖是 `requests`：

```bash
pip3 install requests
```

如果后续项目增加 `requirements.txt`，建议改为：

```bash
pip3 install -r requirements.txt
```

### 4.2 克隆或更新项目

如果服务器还没有项目目录：

```bash
sudo mkdir -p /var/www
sudo chown ubuntu:ubuntu /var/www
cd /var/www
git clone https://github.com/chenfeng2498/DAA.git 大A监控
```

如果目录已经存在：

```bash
cd /var/www/大A监控
git pull origin master
```

### 4.3 启动服务的正确方式

推荐使用 `systemd` 管理服务，不推荐长期使用 `nohup`。`systemd` 可以支持开机自启、异常自动重启、统一查看日志。

创建服务文件：

```bash
sudo tee /etc/systemd/system/daa-monitor.service > /dev/null <<'EOF'
[Unit]
Description=DAA Monitor Backend
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/var/www/大A监控
ExecStart=/usr/bin/python3 /var/www/大A监控/backend/server.py
Restart=always
RestartSec=3
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
EOF
```

启用并启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable daa-monitor
sudo systemctl restart daa-monitor
```

查看服务状态：

```bash
sudo systemctl status daa-monitor --no-pager
```

## 5. 端口与防火墙

当前涉及两个端口：

| 端口 | 用途 | 访问方 |
| :--- | :--- | :--- |
| `8080` | 大A监控 Web/API 服务 | 浏览器、前端页面、本地测试 |
| `5566` | Go-WXPush 微信推送服务 | 大A监控后端、本地联调测试 |

大A监控服务监听端口为 `8080`。

服务器内部防火墙 `ufw` 当前状态曾确认是 `inactive`。如需手动放行：

```bash
sudo ufw allow 8080/tcp
sudo ufw allow 5566/tcp
sudo ufw status verbose
```

腾讯云轻量应用服务器还需要在控制台放行端口。

大A监控 Web/API 服务：

| 项目 | 值 |
| :--- | :--- |
| 方向 | 入站 |
| 协议 | TCP |
| 端口 | 8080 |
| 来源 | `0.0.0.0/0` |

Go-WXPush 微信推送服务：

| 项目 | 值 |
| :--- | :--- |
| 方向 | 入站 |
| 协议 | TCP |
| 端口 | 5566 |
| 来源 | `0.0.0.0/0` |

`5566` 当前为了支持本地连接服务器测试和多人联调用 `0.0.0.0/0` 放行。后续如果要收紧暴露面，可以改为只允许大A监控后端服务器公网 IP，或者给 Go-WXPush 增加鉴权后继续公网访问。

如果服务器内部 `curl http://localhost:8080` 正常，但浏览器访问 `http://111.230.16.149:8080` 失败，优先检查腾讯云控制台防火墙是否放行 `8080`。

如果 Go-WXPush 在服务器内部正常监听，但本地访问 `http://111.230.16.149:5566/wxsend` 超时，优先检查腾讯云控制台防火墙是否放行 `5566`。

## 8. GitHub Actions 自动部署

GitHub 仓库：

```text
https://github.com/chenfeng2498/DAA
```

工作流文件：

```text
.github/workflows/deploy.yml
```

当前配置在 `master` 分支 push 后触发，通过 SSH 登录服务器并执行部署脚本。

GitHub Secrets：

| Secret | 说明 |
| :--- | :--- |
| `SERVER_HOST` | 服务器公网 IP，例如 `111.230.16.149` |
| `SERVER_USERNAME` | SSH 用户名，例如 `ubuntu` |
| `SSH_PRIVATE_KEY` | 私钥 `key_fordaa.pem` 的完整内容 |

### 8.1 推荐的 deploy.yml

建议将部署动作改为使用 `systemd` 重启服务，而不是 `nohup`。推荐配置如下：

```yaml
name: Deploy to Server

on:
  push:
    branches: [ master ]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Deploy and Restart Backend
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USERNAME }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script_stop: true
          script: |
            echo "=== 拉取最新代码 ==="
            cd /var/www/大A监控
            git pull origin master
            echo "=== 安装依赖 ==="
            pip3 install requests -q
            echo "=== 重启 systemd 服务 ==="
            sudo systemctl restart daa-monitor
            echo "=== 查看服务状态 ==="
            sudo systemctl status daa-monitor --no-pager
            echo "=== 部署完成 ==="
```

如果 GitHub Actions 执行 `sudo systemctl restart daa-monitor` 时要求密码，需要在服务器配置免密 sudo。

### 8.2 配置 GitHub Actions 免密重启服务

执行：

```bash
sudo visudo
```

追加一行：

```text
ubuntu ALL=(ALL) NOPASSWD: /bin/systemctl restart daa-monitor, /bin/systemctl status daa-monitor
```

如果服务器上的 `systemctl` 路径不同，先查看：

```bash
which systemctl
```

并把 `visudo` 里的 `/bin/systemctl` 替换为实际路径。

## 9. 常见故障处理

### 9.1 浏览器提示“拒绝连接”

可能原因：

- 没有带端口访问，误访问了 `http://111.230.16.149`
- 服务没有启动
- `8080` 端口没有监听
- 腾讯云控制台防火墙未放行 `8080`

排查：

```bash
sudo systemctl status daa-monitor --no-pager
ss -tlnp | grep 8080
curl -s http://localhost:8080/api/dashboard | head
```

如果本机 `curl` 正常，但浏览器访问失败，检查腾讯云控制台防火墙。

### 9.2 `curl -I` 返回 501

执行：

```bash
curl -I http://localhost:8080
```

可能返回：

```text
HTTP/1.0 501 Unsupported method ('HEAD')
```

这不是核心故障。原因是 `backend/server.py` 实现了 `GET`，但没有实现 `HEAD`。浏览器正常访问页面使用 `GET`，不受影响。

正确验证方式：

```bash
curl http://localhost:8080
curl http://localhost:8080/api/dashboard
```

### 9.3 页面打开但没有数据

可能原因：

- 只启动了 `python3 -m http.server 8000`
- 没有启动 `backend/server.py`
- 前端请求 `/api/dashboard`，但当前服务没有提供 API

解决：

```bash
sudo systemctl restart daa-monitor
curl -s http://localhost:8080/api/dashboard | head
```

不要使用下面的方式作为正式线上服务：

```bash
python3 -m http.server 8000
```

### 9.4 systemd 服务启动失败

查看日志：

```bash
journalctl -u daa-monitor -n 100 --no-pager
```

常见原因：

- `requests` 未安装
- `/var/www/大A监控` 路径不存在
- `backend/server.py` 文件不存在
- `8080` 端口已被其他进程占用

检查端口占用：

```bash
ss -tlnp | grep 8080
```

如果有旧进程占用：

```bash
pkill -f "server.py"
sudo systemctl restart daa-monitor
```

### 9.5 GitHub Actions 部署成功但服务没起来

排查：

```bash
cd /var/www/大A监控
git log -1 --oneline
sudo systemctl status daa-monitor --no-pager
journalctl -u daa-monitor -n 100 --no-pager
```

如果 Actions 仍使用 `nohup python3 backend/server.py`，建议改为 `sudo systemctl restart daa-monitor`。

### 9.6 微信提醒发送失败：连接 Go-WXPush 服务超时

现象：

```text
微信提醒发送失败 601012#4: 连接 Go-WXPush 服务超时
```

含义：大A监控后端已经触发价格提醒，但连接 `http://111.230.16.149:5566/wxsend` 超时。提醒不会被标记为已发送，状态仍保持“监测中”，下一轮后台刷新会继续重试。

服务器上先确认 Go-WXPush 端口是否监听：

```bash
ss -lntp | grep 5566
```

正常结果类似：

```text
LISTEN 0 4096 0.0.0.0:5566 0.0.0.0:*
LISTEN 0 4096 [::]:5566 [::]:*
```

再确认服务器本机调用是否正常：

```bash
curl -v --max-time 8 "http://127.0.0.1:5566/wxsend?title=test&content=test"
curl -v --max-time 8 "http://111.230.16.149:5566/wxsend?title=test&content=test"
```

本地 Windows 端验证：

```powershell
Test-NetConnection 111.230.16.149 -Port 5566
python backend\server.py wxpush-check
```

判断方式：

- `127.0.0.1:5566` 都超时：Go-WXPush 服务内部异常，需要重启服务或查看 Go-WXPush 日志。
- `127.0.0.1:5566` 正常，但 `111.230.16.149:5566` 超时：检查 Ubuntu 防火墙或腾讯云轻量应用服务器防火墙是否放行 `TCP 5566`。
- 服务器公网 IP 正常，但本地 Windows 超时：检查本地网络、代理或运营商链路。

当前为了本地联调和多人使用，腾讯云轻量应用服务器防火墙需保留：

| 项目 | 值 |
| :--- | :--- |
| 方向 | 入站 |
| 协议 | TCP |
| 端口 | 5566 |
| 来源 | `0.0.0.0/0` |

大A监控后端启动时会打印当前 Go-WXPush 配置：

```text
Go-WXPush: http://111.230.16.149:5566/wxsend timeout=6s/6s
```

这只表示当前配置，不代表接口已经连通。连通性以 `python backend\server.py wxpush-check` 的结果为准。

## 10. 后续优化建议

### 10.1 使用 Nginx 反向代理 80 端口

当前访问地址需要带端口：

```text
http://111.230.16.149:8080
```

后续可以安装 Nginx，将 `80` 端口转发到 `8080`，这样可以直接访问：

```text
http://111.230.16.149
```

示例配置：

```nginx
server {
    listen 80;
    server_name 111.230.16.149;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 10.2 增加 requirements.txt

建议新增：

```text
requests
```

然后部署脚本改为：

```bash
pip3 install -r requirements.txt
```

### 10.3 增加健康检查接口

可以在 `backend/server.py` 增加：

```text
/api/health
```

返回：

```json
{"ok": true}
```

这样 GitHub Actions、Nginx 或监控工具可以更稳定地判断服务状态。
