# 大A监控 — 需求文档 V1

## 一、项目定位

面向 **不喜欢杂乱信息** 的 A 股投资者，做一个极简的市场监控面板。页面允许纵向滚动，但核心信息层级必须清晰：MA2500 估值区间图是第一核心，自选股关键信息完整保留，避免横向滚动。

## 二、设计原则

- **极简**：只放核心数据，砍掉一切可有可无的指标
- **一目了然**：颜色、位置、数值直观传达多空状态
- **核心优先**：MA2500 估值区间图作为页面主视觉，图形必须足够大，6 条标准估值线、右轴名称与数值标签和背景色带必须清晰可读
- **纵向滚动**：页面允许纵向滚动，不为了塞进一屏牺牲关键信息；禁止出现横向滚动
- **盘中自动刷新**：每 10 秒

## 三、页面布局

当前前端以 `design-prototype.html` 的视觉原型为准，页面按单列纵向主流程排列：顶部状态栏 → 四指数卡片 → MA2500 核心大图 → 市场状态分析 → 涨跌分布 → 自选股卡片网格；底部保留固定一句话总结悬浮条。所有模块共享浅紫页面背景、白色内容面板、统一的标题层级、圆角、边框和阴影风格。

## 四、功能模块

### 4.1 指数总览

4 个核心指数卡片（上证指数、深证成指、创业板指、科创50），每张卡片仅展示两类核心信息：

1. 当日点位与涨跌幅，视觉上突出显示。
2. `MA2500` 估值定位，直接输出文字结论，不展示偏离百分比、不展示历史分位。

定位逻辑：计算 `P / MA2500`，其中 `P` 为当前指数最新点位或最新收盘价，`MA2500` 为该指数自身的 MA2500 中心线。指数卡片严格按下表输出六档结论；下方核心图按 4.2 独立绘制 6 条带文字的估值参考线，其中 `MA1 / 1.35` 仅作为“极度低估”参考线，不新增第七档卡片结论。

| 比值范围（`P / MA2500`） | 输出结论 |
|---|---|
| `>= 1.65` | 玩命 |
| `[1.45, 1.65)` | 高度泡沫 |
| `[1.25, 1.45)` | 轻度泡沫 |
| `[1.05, 1.25)` | 估值合理 |
| `[1 / 1.15, 1.05)` | 比较便宜 |
| `< 1 / 1.15` | 极度低估 |

任何指数的有效历史样本不足 2500 个交易日时，不计算、不绘制 MA2500，也不输出估值结论。指数卡片对应区域显示 `MA2500：数据不足（已有 N/2500 日）`；核心图对应指数样本不足时显示空状态，禁止用成立以来均值代替 MA2500。

展示要求：

- 卡片文案统一使用 `MA2500：估值合理`、`MA2500：轻度泡沫` 这类格式。
- 禁止出现 `vs 十年线`、`十年线偏离`、`偏离 -8.2%`、`历史分位 22%` 等需要用户二次判断的表达。
- 指数卡片只保留 MA2500 六档估值结论，不再同时展示另一套“分位定性标签”。
- 数据不足提示使用信息图标：桌面端支持悬浮和键盘聚焦显示说明，移动端点击图标打开非模态浮层；点击外部、再次点击或按 `Esc` 关闭。图标必须有可访问名称，不能只依赖悬浮操作。

### 4.2 MA2500 估值区间图（核心模块）

MA2500 使用收盘价 2500 个交易日移动平均。只有当目标日期及之前存在连续 2500 条有效交易日收盘数据时才输出 MA2500；不足 2500 条时返回空值，不得使用成立以来均值、固定均值或其他缩短窗口的算法替代。

**准确公式**：

```text
MA1: MA(CLOSE, 2500)
玩命线: MA1 * 1.65
高度泡沫线: MA1 * 1.45
轻度泡沫线: MA1 * 1.25
估值合理线: MA1 * 1.05
比较便宜线: MA1 / 1.15
极度低估线: MA1 / 1.35
```

`MA1` 是统一计算基线，不单独绘制。图表只绘制以下 6 条标准估值线和背景色带；禁止继续使用 `MA1 * 1.2`、`MA1 * 1.4`、`MA1 * 1.6`、`MA1 / 1.2` 等旧口径。

**6 条标准估值线**：

| 标注位置（画线公式） | 文案 |
|----------------------|------|
| `MA1 * 1.65` | 玩命 |
| `MA1 * 1.45` | 高度泡沫 |
| `MA1 * 1.25` | 轻度泡沫 |
| `MA1 * 1.05` | 估值合理 |
| `MA1 / 1.15` | 比较便宜 |
| `MA1 / 1.35` | 极度低估 |

图表展示上证指数走势线，并叠加上述 6 条标准估值线。每条线的右侧直接标注该线对应的文案和当前具体数值。相邻估值线之间填充低饱和透明色带，颜色从图表下方的绿色平滑过渡到上方的红色。最高线上方延续顶部红色色带，最低线下方延续底部绿色色带。

色带仅用于帮助识别指数走势在线条之间的位置，不设置区间名称，不在色带内显示文字，也不根据色带数量定义额外估值档位。页面中的估值文字只允许来自 6 条标准估值线文案和 4.1 定义的六档指数结论。

展示要求：

- MA2500 估值区间图是页面核心模块，应占据主内容区最大面积；模块采用浅紫外层 + 白色图表面板结构，标题栏展示当前指数点位、估值标签和 MA2500 数值。
- 图表画线使用近 20 年上证指数数据；但 MA2500 的计算窗口不能只取图内数据，必须使用图表日期之前的完整历史数据参与 2500 日均线计算。
- 为保证 MA2500 有足够前置历史并支持长周期回溯，指数历史数据同步起点使用 `19900101`；实际最早日期以交易所和数据源返回为准。
- 图内保留 6 条标准估值线及其右侧名称和具体线值，线值严格使用本节公式；所有右轴线值默认保留两位小数。
- 右侧标签使用与市场状态分析一致的彩色数据标签样式，并通过引导线连接对应曲线；标签按纵向位置自动避让，不得互相重叠、被裁切或造成横向滚动。
- 顶部标题栏同步显示当前指数点位、当前 `MA2500` 数值和六档估值标签；标题栏数值必须与图中同一时点数据一致。
- 图内不再额外显示“上证走势”“六条标准估值线”等说明性文字，避免干扰图形阅读。
- 横坐标底部按数据范围动态生成年度刻度。默认近 20 年视图采用每 2 年一个主刻度，例如当前为 2026 年时显示 `2006 / 2008 / ... / 2026`；窄屏可自动降低刻度密度，但不得重叠。
- 顶部摘要信息合并到模块标题栏，保留点位、MA2500 数值和六档估值标签；不展示偏离百分比或历史分位，不在图下重复展示进度条或图例。

### 4.3 市场状态分析

市场状态分析用于回答“当前指数趋势是否健康、趋势动能是否改善”。该模块优先服务大盘状态判断，不替代 MA2500 长周期估值区间图。

展示内容：

| 区域 | 内容 | 说明 |
|------|------|------|
| 顶部切换 | `市场状态分析` 标题 + 指数选择 | 默认上证指数，可扩展深证成指、创业板指等 |
| K线图 | 指数 K 线 + MA250 + MA120 | K 线展示指数价格波动，MA250/MA120 用不同颜色叠加观察中长期趋势位置 |
| MA250 一阶导 | MA250 斜率曲线 | 判断 MA250 本身向上/向下，红色表示斜率 > 0，绿色表示斜率 <= 0 |
| MA250 二阶导 | MA250 斜率变化曲线 | 判断趋势动能改善/恶化；显示 `0` 基准线，可显示 `-4` 个百分点预警参考线 |

计算定义：

```text
MA250_t = MA(CLOSE, 250)
MA120_t = MA(CLOSE, 120)
MA250_slope_t = (MA250_t / MA250_{t-N} - 1) * 100%
MA250_accel_t = MA250_slope_t - MA250_slope_{t-N}
```

其中 `N` 默认取 20 个交易日，用于表示约 1 个月的趋势变化。若后续验证发现过于敏感，可调整为 30 或 60 个交易日。代码中以常量 `TREND_WINDOW = 20` 定义，便于后续调参。

其中 `MA250_accel` 的单位为百分点，例如斜率由 `-2%` 变为 `-1%`，则二阶导为 `+1` 个百分点。`-4` 个百分点参考线只用于提示动能快速恶化，不参与状态分类。

定性规则采用自上而下优先匹配，命中第一条后停止，保证任一有效数据只能输出一个状态：

| 优先级 | 状态 | 条件 | 含义 |
|--------|------|------|------|
| 1 | 高位钝化 | `P >= MA250 * 1.05`、`MA250_slope > 0` 且 `MA250_accel < 0` | 位置较高且长期趋势仍向上，但动能走弱 |
| 2 | 牛市趋势 | `P >= MA250` 且 `MA250_slope > 0` | 指数和长期均线共同向上 |
| 3 | 高位转弱 | `P >= MA250` 且 `MA250_slope <= 0` | 指数仍在长期均线上方，但长期均线已走平或向下 |
| 4 | 修复阶段 | `P < MA250` 且（`MA250_slope > 0` 或 `MA250_accel > 0`） | 指数仍在长期均线下方，但趋势或动能已经改善 |
| 5 | 熊市趋势 | `P < MA250`、`MA250_slope <= 0` 且 `MA250_accel <= 0` | 指数位于长期均线下方，趋势和动能均未改善 |

`P = MA250` 归入上方状态；`MA250_slope = 0` 和 `MA250_accel = 0` 均按非正值处理。MA250、斜率或二阶导任一数据不足时不输出状态，显示“趋势数据不足”。

展示要求：

- 该模块位于 MA2500 核心图与涨跌分布之间，允许纵向滚动。
- 模块顶部为标题和指数选择栏，默认显示“上证指数 - MA250 斜率分析”。
- 模块内部按纵向堆叠三个独立白色圆角面板：顶部 K 线图，中部 MA250 一阶导，底部 MA250 二阶导；不得改为左右并排布局。
- 图表采用清爽浅色背景，网格线低对比度，不使用暗黑交易终端风格。
- K 线、MA250、MA120 使用不同颜色区分；顶部图例标注 `MA250` 和 `MA120`，涨跌 K 线按红涨绿跌显示。
- 顶部 K 线图右侧保留价格坐标和当前值标签；一阶导和二阶导右侧保留百分比坐标、当前值标签、`0` 基准线，二阶导保留 `-4%` 预警线。
- 默认展示最近 3 年数据，保留横轴年份/月度刻度；窄屏按可用宽度降低刻度密度，不得重叠。

### 4.4 涨跌分布

涨跌分布用于集中展示当日全市场涨跌幅结构、市场情绪和成交量状态。市场情绪与成交量不再作为独立卡片展示，统一融合到该模块的顶部摘要区，避免页面信息碎片化。

展示内容：

| 区域 | 内容 |
|------|------|
| 顶部摘要 | 涨跌家数、涨跌比、涨停/跌停数、全A成交额、量能标签、情绪标签 |
| 分桶柱状图 | 按当日涨跌幅统计股票数量，绿色为下跌区间，红色为上涨区间，中性灰为 0 附近 |
| 分桶标签 | `<-10%`、`-10~-7%`、`-7~-5%`、`-5~-3%`、`-3~0%`、`0%`、`0~3%`、`3~5%`、`5~7%`、`7~10%`、`>10%` |
| 汇总文字 | 跌家数、涨家数、跌停家数、涨停家数 |
| 宽度条 | 用一条横向比例条表示上涨/下跌占比，中间可用断点区分 |
| 成交额明细 | 全A、沪市A股、深市A股、北交所、ETF 五类成交额及其相对历史基准的变化 |

布局要求：顶部为模块标题、说明和情绪标签；其下为市场广度、涨跌比、涨停/跌停、成交额/量能摘要；再下方为 11 桶涨跌分布图、横向涨跌占比条和五类成交额明细。市场情绪与成交量不单独拆成页面右侧卡片。

分桶规则：

```text
x < -10%             -> 跌 >10%
-10% <= x < -7%      -> 跌 10~7
-7% <= x < -5%       -> 跌 7~5
-5% <= x < -3%       -> 跌 5~3
-3% <= x < 0%        -> 跌 3~0
x = 0%               -> 平盘
0% < x <= 3%         -> 涨 0~3
3% < x <= 5%         -> 涨 3~5
5% < x <= 7%         -> 涨 5~7
7% < x <= 10%        -> 涨 7~10
x > 10%              -> 涨 >10%
```

说明：

- A 股涨跌停比例存在 10%、20%、ST 5% 等差异，涨跌停家数应优先用实时行情字段判断；若无涨跌停字段，才用涨跌幅阈值近似判断。
- “全市场”在本模块中指沪市、深市和北交所全部 A 股股票，包括主板、科创板、创业板、北交所和 ST 股票；不包括 B 股、ETF/LOF、债券、可转债、期权和其他基金。分桶统计排除停牌、退市整理、无有效昨收价或无有效实时涨跌幅的标的。
- ETF 只在成交额明细中单列展示，不计入全A涨跌家数、涨跌分桶、市场情绪或“全A成交额”。
- 颜色遵循 A 股习惯：红色表示上涨，绿色表示下跌。
- 市场情绪实时数据不可用时，顶部摘要显示“等待实时数据”，分桶图不展示空数据。
- 涨跌幅 `x` 使用数据源提供的未四舍五入原始值判定，只有原始值精确等于 `0` 才计为平盘；显示时可按两位小数格式化。
- 该模块位于市场状态分析附近，作为市场广度、情绪和量能的统一入口。

**市场情绪计算**：

```text
valid_count = up_count + down_count + flat_count
breadth_score = (up_count - down_count) / valid_count * 100
```

| `breadth_score` | 情绪标签 |
|-----------------|----------|
| `>= 40` | 极强 |
| `[15, 40)` | 偏强 |
| `(-15, 15)` | 均衡 |
| `(-40, -15]` | 偏弱 |
| `<= -40` | 极弱 |

边界 `15` 归入“偏强”，`-15` 归入“偏弱”；`valid_count = 0` 时显示“等待实时数据”。涨跌比显示为 `up_count : down_count`，不含平盘；任一侧为 0 时仍显示真实值，例如 `1280 : 0`，不得用除零后的无穷大代替。

**成交额范围与量能标签**：

| 类型 | 统计范围 | 是否计入全A成交额 |
|------|----------|------------------|
| 沪市A股 | 上交所主板 + 科创板 A 股 | 是 |
| 深市A股 | 深交所主板 + 创业板 A 股 | 是 |
| 北交所 | 北交所股票 | 是 |
| 全A | 沪市A股 + 深市A股 + 北交所 | - |
| ETF | 沪深北交易所 ETF，单独统计 | 否 |

成交额直接汇总实时行情列表中各证券的当日累计成交额，不使用指数成交额相加代替。每一类型分别计算量能比：

```text
amount_ratio = 当前累计成交额 / 最近 20 个交易日同一交易时刻累计成交额的中位数
```

| `amount_ratio` | 量能标签 |
|----------------|----------|
| `>= 1.50` | 极度活跃 |
| `[1.20, 1.50)` | 活跃 |
| `[0.80, 1.20)` | 正常 |
| `[0.60, 0.80)` | 低迷 |
| `< 0.60` | 极度低迷 |

盘中按交易所时间对齐到最近一个完整分钟，使用此前 20 个有效交易日该分钟的累计成交额中位数；收盘后使用此前 20 个完整交易日的全日成交额中位数。历史同分钟样本不足 10 日时不输出量能标签，显示“基准数据不足”。顶部主量能标签使用“全A”结果，其余类型展示成交额、相对基准变化百分比和各自标签。

系统每分钟将五类市场的累计成交额写入 `market_amount_snapshot`，至少保留最近 30 个交易日，字段包括交易日期、交易分钟、市场类型、累计成交额和数据时间戳。首次部署没有历史同分钟快照时允许补抓数据源可提供的历史分钟成交额；无法补抓时进入冷启动，累计满 10 个有效交易日前只展示成交额和“基准数据不足”，不得用不同时间点或简单按交易分钟线性折算的数据代替。

### 4.5 个股监控（自选股卡片网格）

#### 4.5.3 股价分位

当前股价历史分位用于回答“这只股票现在自己的位置高不高”。

计算方式：

```text
price_percentile = 历史前复权收盘价 <= 当前前复权口径价格 的交易日数量 / 历史有效交易日数量 * 100%
```

定性规则：`[0%, 10%)` 极度低估，`[10%, 30%)` 低估，`[30%, 70%)` 合理，`[70%, 90%)` 高估，`[90%, 100%]` 极度高估。

#### 4.5.4 股价大盘映射（核心算法）

目标：回答“当前股价处于该股完整历史价格分布的什么位置；相同历史分位在大盘点位分布中对应多少点；当前大盘与该位置相比是高还是低”。

个股映射大盘采用 **历史分位映射法**。先计算当前股价在该股完整前复权历史价格中的分位，再从相同交易日集合的上证指数点位中取相同分位数。该方法比较的是个股与大盘各自完整历史分布中的相对位置，不使用个股价格与指数点位的比值。

输入变量：

```text
P_t：个股在交易日 t 的前复权收盘价
I_t：大盘指数在交易日 t 的收盘点位，默认使用上证指数 000001
T：个股上市首日至当前最新交易日，且个股和指数同时存在数据的交易日集合
P_current_adj：当前股价对应的前复权价格；盘中实时价按最新复权因子转换到历史前复权口径
I_current：上证指数当前有效点位，盘中优先使用实时点位
```

计算步骤：

```text
stock_percentile = Count(P_t <= P_current_adj) / Count(T) * 100%
I_mapped = Quantile({I_t | t ∈ T}, stock_percentile)
index_percentile = Count(I_t <= I_current) / Count(T) * 100%
Position_gap = index_percentile - stock_percentile
```

`Quantile` 使用最近秩法：将有效 `I_t` 升序排列，样本数为 `n`，分位值 `q = stock_percentile / 100`，取第 `max(1, ceil(q * n))` 个值。该定义必须由前后端共用，避免不同插值算法产生不一致结果。

判定逻辑：

| 分位差 | 含义 | 页面展示 |
|--------|------|----------|
| `Position_gap > 5` 个百分点 | 当前大盘的历史位置高于该股当前价格位置 | 大盘位置更高 / `↑` |
| `Position_gap < -5` 个百分点 | 当前大盘的历史位置低于该股当前价格位置 | 大盘位置更低 / `↓` |
| `-5 <= Position_gap <= 5` 个百分点 | 当前大盘与该股处于相近历史位置 | 位置相近 / `≈` |

示例：

```text
隆基绿能当前前复权价格在完整有效历史中处于 18% 分位。
同一交易日集合中，上证指数 18% 分位点位 I_mapped = 2782 点。
当前上证指数在该集合中处于 72% 分位。
Position_gap = 72 - 18 = 54 个百分点。
结论：当前大盘的历史位置明显高于该股当前价格位置。
```

数据处理要求：

- 个股历史价格必须使用前复权收盘价，避免分红送股导致价格断层。
- 个股和大盘指数按交易日内连接，只保留二者同时存在数据的日期；个股停牌日自动剔除。
- 历史范围使用个股上市以来的全部有效交易日，不使用固定近 1 年、近 5 年等截断窗口。
- 有效样本数少于 250 个交易日时不计算，页面显示“历史样本不足（N/250 日）”。
- 若当前实时价无法可靠转换为前复权口径，则回退到最近一个有效前复权收盘价，并标注数据日期；禁止将未复权实时价直接与前复权历史价格比较。
- 卡片优先展示个股 `stock_percentile`、映射点位 `I_mapped`、当前上证 `index_percentile` 和 `Position_gap` 结论。分位显示为整数百分比，分位差显示为整数个百分点。

### 4.7 实时价格与刷新口径

价格字段按以下优先级选择，保证盘中尚无当日收盘价时仍能展示当前状态：

| 场景 | 指数/个股当前价格 `P` | 涨跌幅与成交额 | 页面时间标识 |
|------|-----------------------|----------------|--------------|
| 交易日盘中 | 东方财富实时行情最新价；最新价无效时回退 DB 最近收盘价 | 使用实时累计值；无效时显示等待实时数据 | `实时 HH:mm:ss` 或 `截至 HH:mm:ss` |
| 当日收盘后且实时行情已更新 | 当日最终实时价；与日线收盘一致后优先使用当日日线 | 使用当日最终值 | `收盘 YYYY-MM-DD` |
| 午间休市 | 上午收盘时的最新实时价 | 使用上午累计值 | `午间休市 HH:mm:ss` |
| 非交易日 | DB 最近一个有效交易日收盘价 | 使用最近交易日最终值，不继续自动刷新 | `收盘 YYYY-MM-DD` |
| 实时接口不可用 | DB 最近一个有效交易日收盘价 | 实时字段显示“等待实时数据” | `数据截至 YYYY-MM-DD` |

交易时段按交易所日历判断，不用本地星期简单推断。A 股常规连续竞价时段为 `09:30-11:30`、`13:00-15:00`；集合竞价期间可显示实时行情，但不参与每 10 秒自动刷新规则，除非数据源能够稳定提供有效最新价。

盘中每 10 秒只刷新实时行情、指数当前估值结论、基于实时点位的市场状态结论、涨跌分布、市场情绪、成交额和一句话总结；MA2500 历史序列、MA250/MA120 历史序列、个股完整历史分位基准仅在启动、添加自选股、手动重新加载或发现新交易日日线后重算。页面请求并发时保留最后一次成功数据，较早请求的响应不得覆盖较新响应。

实时接口返回值必须同时通过证券代码匹配、时间戳有效、最新价大于 0 三项校验。接口超时、字段缺失、价格为 0 或时间戳早于当前有效交易日时视为无效并执行上述回退，不得把占位值当作真实行情。

## 六、数据源与计算

- 历史日线：东方财富 kline API，存 SQLite `index_daily` / `stock_daily`
- 实时行情：东方财富 quote API 和全市场行情列表 API；按 4.7 的价格优先级、时间校验和回退规则使用，网络不可用时不阻塞历史数据展示
- 实时字段缩放：个股 quote 字段 `f43` 当前价、`f170` 涨跌幅、`f115` PE、`f167/f117` PB、`f162` PE 分位按接口值 / 100 使用；市值字段按元 / 1e8 转为亿元
- MA2500：SQLite 窗口函数 `AVG() OVER (ROWS 2499 PRECEDING)`；只有 `COUNT() = 2500` 时输出结果，样本不足时返回空值并按 4.1 显示进度，不得使用缩短窗口兜底
- MA2500 指数卡片结论计算口径：四指数卡片中的 `MA2500` 估值定位结论，必须与下方 MA2500 核心图的 `MA1` 数值使用同一数据源、同一计算窗口（2500 个交易日）。禁止单独拉取不同时间段的数据计算，确保卡片结论与图形展示完全对应。结论输出时，只输出 4.1 中定义的 6 种文字标签之一，不附加任何百分比数值。
- 市场状态分析：指数 OHLC/K 线、MA250、MA120、MA250 一阶导/二阶导均基于 `index_daily` 历史日线计算
- 涨跌分布：使用东方财富全市场实时行情列表，按 4.4 定义的全A范围统计当日涨跌幅、涨跌停状态、涨跌家数和停牌状态；实时接口不可用时显示“等待实时数据”
- 成交额与量能：从证券级实时行情汇总全A、沪市A股、深市A股、北交所和 ETF 成交额，并与最近 20 个交易日同一时刻累计成交额中位数比较，按 4.4 输出五档量能标签
- 成交额分钟基准：每分钟写入 SQLite `market_amount_snapshot`，至少保留最近 30 个交易日；历史分钟数据无法补抓时按 4.4 的冷启动规则展示
- 股价分位：基于个股上市以来全部有效前复权收盘价计算；大盘映射：按 4.5.4 使用历史分位映射法，将个股当前价格分位映射到同一交易日集合的上证指数同分位点位
